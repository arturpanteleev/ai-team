## ДОБАВЛЕННЫЕ Требования

### Требование: Deployer коммитит и создаёт PR
Агент Deployer ДОЛЖЕН закоммитить изменения и создать pull request.

#### Сценарий: Успешный деплой
- **КОГДА** Deployer запускается
- **И** вердикт review.md — APPROVED
- **И** статус test-report.md — PASS
- **ТОГДА** Deployer ДОЛЖЕН:
  - Заддить все изменения через `git add`
  - Закоммитить с описательным сообщением, включающим имя фичи и задачи
  - Запушить в удалённый репозиторий
  - Создать PR с заголовком и описанием из proposal.md

#### Сценарий: Проверки не пройдены
- **КОГДА** review.md не APPROVED
- **ИЛИ** test-report.md не PASS
- **ТОГДА** Deployer ДОЛЖЕН прерваться
- **И** вывести ошибку: "Ревью или тесты не пройдены. Исправьте проблемы и запустите заново."

### Требование: Формат сообщения коммита
Сообщение коммита ДОЛЖНО включать имя фичи и ссылку на артефакты.

#### Сценарий: Формат коммита
- **КОГДА** Deployer коммитит
- **ТОГДА** сообщение коммита ДОЛЖНО быть: `feat: {feature} — {краткое описание задачи}\n\nRef: .ai-team/artifacts/{feature}/`
