## ДОБАВЛЕННЫЕ Требования

### Требование: Порядок выполнения пайплайна
Система ДОЛЖНА выполнять агентов в строгом порядке: Analyst → Architect → Coder → Reviewer → Tester → Deployer.

#### Сценарий: Последовательное выполнение
- **КОГДА** пайплайн запускается
- **ТОГДА** Analyst выполняется первым
- **И** Architect выполняется только после успешного завершения Analyst
- **И** Coder выполняется только после успешного завершения Architect
- **И** каждый последующий агент ждёт завершения предыдущего

#### Сценарий: Сбой пайплайна
- **КОГДА** любой агент возвращает ошибку
- **ТОГДА** пайплайн ДОЛЖЕН остановиться
- **И** последующие агенты НЕ ДОЛЖНЫ выполняться

### Требование: Коммуникация через артефакты
Агенты ДОЛЖНЫ общаться через файлы в `.ai-team/artifacts/`.

#### Сценарий: Агент читает входные данные
- **КОГДА** агент запускается
- **ТОГДА** система ДОЛЖНА прочитать пути его входных артефактов из `def.yaml`
- **И** передать их содержимое как контекст в runtime

#### Сценарий: Агент записывает выходные данные
- **КОГДА** агент успешно завершается
- **ТОГДА** система ДОЛЖНА проверить, что его выходные артефакты существуют
- **И** пометить их как готовые для следующего агента

### Требование: Изоляция фич
Каждая фича ДОЛЖНА иметь своё дерево артефактов в `.ai-team/artifacts/{feature}/`.

#### Сценарий: Несколько фич
- **КОГДА** пользователь запускает пайплайн для "add-auth", а позже для "add-payment"
- **ТОГДА** артефакты ДОЛЖНЫ храниться раздельно: `.ai-team/artifacts/add-auth/` и `.ai-team/artifacts/add-payment/`

### Требование: Настройка пайплайна
Пайплайн ДОЛЖЕН настраиваться через `.ai-team/config.yaml`.

#### Сценарий: Кастомный пайплайн
- **КОГДА** пользователь указывает `pipeline: [analyst, coder, deployer]` в конфиге
- **ТОГДА** система ДОЛЖНА выполнить только указанных агентов в заданном порядке
