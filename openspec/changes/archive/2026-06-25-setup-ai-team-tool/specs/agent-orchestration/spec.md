## ДОБАВЛЕННЫЕ Требования

### Requirement: Порядок выполнения пайплайна
Система MUST выполнять агентов в строгом порядке: Analyst → Architect → Coder → Reviewer → Tester → Deployer.

#### Scenario: Последовательное выполнение
- **КОГДА** пайплайн запускается
- **ТОГДА** Analyst выполняется первым
- **И** Architect выполняется только после успешного завершения Analyst
- **И** Coder выполняется только после успешного завершения Architect
- **И** каждый последующий агент ждёт завершения предыдущего

#### Scenario: Сбой пайплайна
- **КОГДА** любой агент возвращает ошибку
- **ТОГДА** пайплайн MUST остановиться
- **И** последующие агенты MUST NOT выполняться

### Requirement: Коммуникация через артефакты
Агенты MUST общаться через файлы в `.ai-team/artifacts/`.

#### Scenario: Агент читает входные данные
- **КОГДА** агент запускается
- **ТОГДА** система MUST прочитать пути его входных артефактов из `def.yaml`
- **И** передать их содержимое как контекст в runtime

#### Scenario: Агент записывает выходные данные
- **КОГДА** агент успешно завершается
- **ТОГДА** система MUST проверить, что его выходные артефакты существуют
- **И** пометить их как готовые для следующего агента

### Requirement: Изоляция фич
Каждая фича MUST иметь своё дерево артефактов в `.ai-team/artifacts/{feature}/`.

#### Scenario: Несколько фич
- **КОГДА** пользователь запускает пайплайн для "add-auth", а позже для "add-payment"
- **ТОГДА** артефакты MUST храниться раздельно: `.ai-team/artifacts/add-auth/` и `.ai-team/artifacts/add-payment/`

### Requirement: Настройка пайплайна
Пайплайн MUST настраиваться через `.ai-team/config.yaml`.

#### Scenario: Кастомный пайплайн
- **КОГДА** пользователь указывает `pipeline: [analyst, coder, deployer]` в конфиге
- **ТОГДА** система MUST выполнить только указанных агентов в заданном порядке
