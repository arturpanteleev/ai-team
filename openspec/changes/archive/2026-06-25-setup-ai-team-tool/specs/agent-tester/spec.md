## ДОБАВЛЕННЫЕ Требования

### Требование: Tester проверяет код на соответствие продуктовой спецификации
Агент Tester ДОЛЖЕН прочитать продуктовую спецификацию и написать/запустить тесты.

#### Сценарий: Tester создаёт отчёт тестирования
- **КОГДА** Tester запускается
- **ТОГДА** он ДОЛЖЕН прочитать `.ai-team/artifacts/{feature}/specs/`
- **И** прочитать `.ai-team/artifacts/{feature}/review.md`
- **И** написать интеграционные/E2E тесты для фичи
- **И** создать `.ai-team/artifacts/{feature}/test-report.md`

### Требование: Содержимое отчёта тестирования
Отчёт тестирования ДОЛЖЕН документировать, какие критерии приёмки проходят, а какие нет.

#### Сценарий: Формат отчёта тестирования
- **КОГДА** отчёт тестирования создан
- **ТОГДА** он ДОЛЖЕН содержать:
  - Список критериев приёмки из спецификации
  - PASS/FAIL для каждого критерия
  - Подтверждение (вывод тестов, логи)
  - Общий статус: PASS / FAIL

### Требование: Ворота пайплайна
Пайплайн ДОЛЖЕН проверять отчёт тестирования перед переходом к Deployer.

#### Сценарий: Тесты не прошли
- **КОГДА** Tester сообщает FAIL
- **ТОГДА** пайплайн ДОЛЖЕН остановиться
- **И** НЕ переходить к Deployer
