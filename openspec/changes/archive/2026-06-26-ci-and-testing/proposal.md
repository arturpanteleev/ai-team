## Зачем

Без CI и тестов нельзя быть уверенным, что изменения не сломают ai-team. Нужен GitHub Actions, автоматические E2E-тесты с mock opencode, и возможность оценивать качество работы агентов (evals).

## Что меняется

- Добавляется `.github/workflows/ci.yaml` — сборка, тесты, vet на каждый push/PR
- Создаётся `testdata/mock-opencode.sh` — заглушка opencode для E2E-тестов
- Добавляется E2E-тест на полный пайплайн с моком
- Создаётся `pkg/eval/` — пакет для оценки качества агентов
- Расширяется CLI командой `ai-team eval`
- В `testdata/` добавляются конфиги для тестовых сценариев (review REJECTED, тесты FAIL и т.д.)

## Возможности

### Новые возможности
- `ci-pipeline`: GitHub Actions workflows
- `mock-opencode`: заглушка для E2E-тестирования без реального AI
- `e2e-testing`: полный пайплайн через mock + верификация артефактов
- `agent-eval`: оценка качества output-артефактов агентов

### Изменённые возможности
- `cli-interface`: новая команда `eval`

## Влияние

- Добавляются зависимости: `github.com/google/go-cmp` (для тестов)
- Mock opencode не должен попасть в PATH при реальном использовании
- Eval-пакет опционален, без внешних зависимостей
