## ADDED Requirements

### Requirement: Поле transition в конфигурации агента
Каждый агент в `pipeline` конфига MUST поддерживать поле `transition` со значениями `auto` или `by_confirm`.

#### Scenario: По умолчанию auto
- **КОГДА** `transition` не указан в конфигурации агента
- **ТОГДА** система MUST использовать значение `auto`

#### Scenario: by_confirm останавливает пайплайн
- **КОГДА** агент с `transition: by_confirm` завершается успешно
- **ТОГДА** система MUST вывести в консоль: `Continue to <next-agent>? [Y/n/diff/summary]`
- **И** MUST ожидать ввод от пользователя

### Requirement: Варианты ответа при confirm
Пользователь MUST иметь возможность выбрать вариант проMUSTия.

#### Scenario: Y — продолжить
- **КОГДА** пользователь вводит `Y` или пустую строку
- **ТОГДА** пайплайн MUST продолжить к следующему агенту

#### Scenario: n — остановить
- **КОГДА** пользователь вводит `n`
- **ТОГДА** пайплайн MUST завершиться досрочно
- **И** MUST быть сгенерирован итоговый отчёт

#### Scenario: diff — показать изменения
- **КОГДА** пользователь вводит `diff`
- **ТОГДА** система MUSTА выполнить `git diff` в targetDir
- **И** показать вывод пользователю
- **И** снова показать приглашение `Continue to <next-agent>? [Y/n/diff/summary]`

#### Scenario: summary — показать сводку этапа
- **КОГДА** пользователь вводит `summary`
- **ТОГДА** система MUSTА показать `.stage-summary/{agent}.md` если он существует
- **И** снова показать приглашение

### Requirement: Неинтерактивный режим
Если stdin не является терминалом, `by_confirm` MUST работать как `auto`.

#### Scenario: CI/pipe
- **КОГДА** `ai-team run` запущен с перенаправленным stdin (pipe)
- **ТОГДА** все `by_confirm` этапы MUST выполняться без остановки
- **И** пользователь MUST NOT получить сообщение с ожиданием ввода
