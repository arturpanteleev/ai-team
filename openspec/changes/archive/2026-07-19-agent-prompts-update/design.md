## Context

Текущие промпты агентов (`agents/*/prompt.md`) содержат минимальные инструкции:
- **analyst** (14 строк): "создать продуктовую спецификацию" с перечислением что должно быть
- **architect** (15 строк): "создать технический дизайн" с перечислением компонентов
- **deployer** (13 строк): "закоммитить и создать PR" с базовыми шагами

Промпты не содержат:
- Строгих правил поведения (принцип минимального изменения, запрет додумывания)
- Требований к формату артефактов
- Ограничений на формат коммитов/PR
- Механизма блокировки при противоречиях
- Требования на русский язык

## Goals / Non-Goals

**Goals:**
- Обновить промпт analyst: добавить требования к proposal.md (бизнес-проблема, scope/out-of-scope, AC с категориями), принцип минимального изменения, запрет додумывания, BLOCKED при противоречиях
- Обновить промпт architect: добавить требования к design.md (затронутые компоненты, контракты, риски, порядок реализации), обнаружение ошибок в требованиях
- Обновить промпт deployer: добавить ограничения на коммиты (≤10 слов, номер задачи, без атрибуции) и PR (≤700 символов)
- Добавить требование на русский язык для всех артефактов

**Non-Goals:**
- Изменение структуры `def.yaml` (inputs/outputs остаются прежними)
- Изменение runtime логики (агенты по-прежнему вызываются через opencode)
- Добавление новых агентов (это Change 3)

## Decisions

### Decision 1: Промпты как единичные markdown файлы

**Выбор:** Перезаписать `prompt.md` для каждого агента целиком.

**Альтернативы:**
- Добавить секции в существующие промпты — промпты слишком короткие, проще переписать
- Вынести правила в отдельный файл — усложняет передачу в runtime

**Rationale:** Перезапись проще и даёт полный контроль над содержимым. Runtime уже поддерживает произвольный промпт из файла.

### Decision 2: Единый набор правил для всех агентов

**Выбор:** Добавить общую секцию "Общие принципы" в каждый промпт.

**Альтернативы:**
- Вынести в отдельный файл и добавлять через runtime — требует изменения runtime
- Добавить только в analyst — непоследовательно

**Rationale:** Каждый агент должен знать общие правила (минимальное изменение, русский язык, BLOCKED). Это можно сделать через копирование секции в каждый промпт.

### Decision 3: Строгие ограничения для deployer

**Выбор:** Зафиксировать формат коммитов и PR в промпте deployer.

**Альтернативы:**
- Ограничить в pipeline logic — не даёт гибкости
- Ограничить в git hook — требует дополнительной настройки

**Rationale:** Промпт deployer — единственное место, где LLM принимает решения о формате. Строгие инструкции в промпте обеспечивают предсказуемость.

## Risks / Trade-offs

- **[Risk]** LLM может проигнорировать строгие правила в промпте → **Mitigation:** Правила сформулированы как imperative ("Ты ДОЛЖЕН", "Запрещено"), что повышает compliance
- **[Risk]** Дублирование общих правил в каждом промпте → **Mitigation:** Правила короткие (5-7 строк), дублирование приемлемо для надёжности
