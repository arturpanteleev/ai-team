## ADDED Requirements

### Requirement: Генерация HTML-отчётов в пайплайне
Пайплайн MUST генерировать HTML-отчёты после каждого этапа и итоговый сводный отчёт.

#### Scenario: Отчёт генерируется после каждого агента
- **КОГДА** агент завершает выполнение (успешно или с ошибкой)
- **ТОГДА** пайплайн MUST вызвать генерацию HTML-отчёта для этого агента
- **И** передать в отчёт: статус, время, входы, выходы, ошибку (если есть)

#### Scenario: Итоговый отчёт в конце
- **КОГДА** пайплайн завершается
- **ТОГДА** пайплайн MUST вызвать генерацию итогового сводного HTML-отчёта
- **И** отчёт MUST содержать сводку по всем этапам с навигацией

### Requirement: Уведомления в пайплайне
Пайплайн MUST вызывать Notifier после каждого завершённого этапа.

#### Scenario: Notify вызывается после каждого этапа
- **КОГДА** агент завершает выполнение
- **ТОГДА** пайплайн MUST вызвать `Notifier.Notify()` для всех зарегистрированных notifier-ов
- **И** передать StageResult: имя агента, статус, сообщение, время

### Requirement: Прогресс-бар в пайплайне
Пайплайн MUST обновлять строку состояния перед и после каждого этапа.

#### Scenario: Обновление прогресс-бара
- **КОГДА** пайплайн начинает выполнение агента
- **ТОГДА** прогресс-бар MUST отобразить имя проекта, фичи и текущего агента
- **И** после завершения агента прогресс-бар MUST обновиться

### Requirement: Ролевая конфигурация в пайплайне
Пайплайн MUST использовать индивидуальные настройки (model, effort, cli) для каждого агента.

#### Scenario: Агент запускается с индивидуальными настройками
- **КОГДА** config.yaml содержит model/cli/effort для конкретного агента
- **ТОГДА** пайплайн MUST передать эти настройки при запуске агента
- **И** MUST NOT использовать глобальные значения для этого агента

### Requirement: Retry-from в пайплайне
Пайплайн MUST поддерживать частичный запуск, начиная с указанного агента.

#### Scenario: Пропуск этапов при retry-from
- **КОГДА** пайплайн запущен с флагом `--retry-from <agent>`
- **ТОГДА** пайплайн MUST пропустить всех агентов до указанного
- **И** проверить наличие артефактов пропущенных этапов
- **И** ЕСЛИ артефакты отсутствуют — вывести ошибку и остановиться

## MODIFIED Requirements

### Requirement: Порядок выполнения пайплайна
Система MUST выполнять агентов в строгом порядке, указанном в конфигурации пайплайна.

#### Scenario: Последовательное выполнение
- **КОГДА** пайплайн запускается (полный запуск)
- **ТОГДА** первый агент в конфигурации выполняется первым
- **И** каждый последующий агент выполняется только после успешного завершения предыдущего
- **И** агенты из конфигурации (переопределяемые через `pipeline:`) MUST выполняться в заданном порядке

#### Scenario: Сбой пайплайна
- **КОГДА** любой агент возвращает ошибку
- **И** НЕ указан флаг `--retry-from`
- **ТОГДА** пайплайн MUST остановиться
- **И** последующие агенты MUST NOT выполняться
- **И** MUST быть сгенерирован итоговый отчёт с указанием упавшего этапа
