## ADDED Requirements

### Requirement: Ролевая конфигурация в config.yaml
`config.yaml` MUST поддерживать переопределение model, effort, cli на уровне каждого агента в пайплайне.

#### Scenario: Парсинг нового формата pipeline
- **КОГДА** `config.yaml` содержит `pipeline` как массив объектов с полем `name`
- **ТОГДА** каждый элемент MUST парситься как структура `AgentConfig` с полями: `name` (обязательно), `model`, `effort`, `cli`
- **И** если поля `model`, `effort`, `cli` не заданы — MUST использоваться глобальные значения

#### Scenario: Обратная совместимость старого формата
- **КОГДА** `config.yaml` содержит `pipeline` как массив строк (например `[analyst, coder]`)
- **ТОГДА** каждая строка MUST интерпретироваться как имя агента
- **И** model/effort/cli для каждого агента MUST браться из глобальных полей конфига

### Requirement: Передача ролевой конфигурации в пайплайн
Система MUST передавать индивидуальные настройки (model, effort, cli) при запуске каждого агента.

#### Scenario: Запуск агента с индивидуальной моделью
- **КОГДА** конфиг coder-а указывает `model: claude-opus-4-20250514`
- **ТОГДА** пайплайн MUST запустить coder с указанной моделью
- **И** это MUST NOT влиять на модель других агентов

### Requirement: Effort per agent
Система MUST поддерживать три уровня effort: `low`, `medium`, `high`.

#### Scenario: Передача effort агенту
- **КОГДА** конфиг агента указывает `effort: high`
- **ТОГДА** пайплайн MUST передать `effort=high` в CLI аргументы агента
- **И** если effort не указан — MUST использоваться глобальное значение или `medium` по умолчанию
