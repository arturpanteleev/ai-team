## ADDED Requirements

### Requirement: Таймаут этапа
Пайплайн MUST ограничивать время выполнения каждого этапа.

#### Scenario: Глобальный таймаут
- **КОГДА** config.yaml содержит `stage_timeout: 30m`
- **И** агент выполняется дольше 30 минут
- **ТОГДА** этап MUST завершиться ошибкой `этап {agent} превысил таймаут 30m`
- **И** пайплайн остановиться

#### Scenario: Per-agent override
- **КОГДА** у агента задан `timeout: 45m`
- **ТОГДА** для него MUST действовать 45m вместо глобального

#### Scenario: Таймаут не задан
- **КОГДА** `stage_timeout` отсутствует в конфиге и per-agent `timeout` не задан
- **ТОГДА** этап выполняется без ограничения времени (обратная совместимость)

### Requirement: Обработка прерывания
Пайплайн MUST корректно завершаться по SIGINT/SIGTERM.

#### Scenario: Ctrl+C во время этапа
- **КОГДА** пользователь прерывает `ai-team run`
- **ТОГДА** текущий агент MUST быть остановлен
- **И** итоговый отчёт и summary MUST быть сгенерированы по выполненным этапам

### Requirement: Валидация артефактов пропущенных этапов при retry-from
При `--retry-from` система MUST проверять выходные артефакты каждого пропускаемого агента.

#### Scenario: Отсутствуют артефакты пропущенного этапа
- **КОГДА** пользователь запускает `--retry-from coder`, а выходы architect отсутствуют
- **ТОГДА** система MUST завершиться с ошибкой `missing artifacts from previous stage: architect`
- **И** MUST NOT запускать агентов
