## MODIFIED Requirements

### Requirement: Точка входа CLI
Система MUST предоставлять единый CLI-бинарник `ai-team` с подкомандами и корректной справкой.

#### Scenario: Вывод справки
- **КОГДА** пользователь запускает `ai-team --help`, `ai-team -h`, `ai-team help` или `ai-team` без аргументов
- **ТОГДА** система MUST вывести информацию об использовании со списком ВСЕХ команд (init, run, list, eval, web, version) и флагов (включая `--retry-from`)
- **И** завершиться с кодом 0 (для явного запроса справки)

#### Scenario: Неизвестная команда
- **КОГДА** пользователь запускает `ai-team <неизвестная-команда>`
- **ТОГДА** система MUST вывести сообщение об ошибке и завершиться с ненулевым кодом

## ADDED Requirements

### Requirement: Exit-коды пайплайна
Команда `run` MUST возвращать различимые exit-коды.

#### Scenario: Таблица кодов
- **КОГДА** `ai-team run` завершается
- **ТОГДА** exit-код MUST быть: `0` — успех; `1` — ошибка этапа/инфраструктуры или негативный вердикт; `2` — BLOCKED; `3` — пользователь остановил пайплайн на gate/confirm

### Requirement: Валидация имени фичи
Система MUST валидировать `--feature` до запуска.

#### Scenario: Недопустимое имя
- **КОГДА** `--feature` содержит `/`, `\`, `..` или пусто
- **ТОГДА** система MUST завершиться с ошибкой `недопустимое имя фичи`
- **И** НЕ создавать файлов
