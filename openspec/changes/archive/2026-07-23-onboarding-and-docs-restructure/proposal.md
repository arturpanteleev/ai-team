## Why

Независимый аудит (`INDEPENDENT_AUDIT_2026-07-23.md`, раздел Developer
Experience) и прямой отзыв пользователя сходятся в одном: `README.md` смешивает
четыре разные аудитории в одном линейном документе — новый пользователь,
впервые устанавливающий инструмент; инженер, проводящий первую фичу через
pipeline; читатель, которому интересно внутреннее устройство delivery/evidence;
контрибьютор, которому нужен процесс разработки самого ai-team. Это создаёт
конкретные проблемы:

- Нет объяснения, как установить и настроить OpenCode CLI и `gh` — оба
  обязательные внешние зависимости, но README упоминает только "OpenCode в
  PATH" без единого слова про установку или авторизацию `gh`.
- Термины (`checkpoint`, `mutation scope`, `candidate`, `canonical delivery
  plan`, `verdict marker`, `BLOCKED`) используются в Quickstart до того, как
  где-либо определены.
- Нет единого пошагового объяснения полного пути фичи через два прохода `run`
  (до и после `--approve-plan`) — эта механика жизненно важна для первого
  успешного запуска, но раньше была разбросана по отдельным примерам без
  связного объяснения "зачем два раза".
- Процесс разработки самого ai-team (OpenSpec-цикл) документирован только в
  `CLAUDE.md` — файле инструкций для Claude Code как агента, не в
  человеко-читаемом contributor guide.

## What Changes

- `README.md` переструктурирован в линейный onboarding-путь: что это / для
  кого (и для кого не) / предварительные требования и установка / быстрый
  старт / как поставить фичу целиком / CLI-справочник / конвейер и зоны
  ответственности (кратко) / конфигурация / evals / глоссарий / граница
  безопасности (кратко, ссылка на AUDIT.md) / разработка (ссылка на
  CONTRIBUTING.md).
- Новый `docs/ARCHITECTURE.md` — глубокое техническое описание
  (детерминированный контроль, evidence layout, deployer/canonical delivery
  plan внутренности, layered agent registry), перенесённое из README без
  изменения технических утверждений.
- Новый `CONTRIBUTING.md` — человеко-читаемое описание OpenSpec-цикла
  разработки ai-team (explore → propose → design → specs → tasks → apply →
  archive), gate rule, make-таргеты, как добавить built-in агента.
- Новый `docs/docs_test.go` — regression guard: проверяет, что README содержит
  обязательные секции и термины глоссария, что ARCHITECTURE.md содержит
  заголовки, на которые ссылается README, и что CONTRIBUTING.md покрывает
  стадии цикла.
- Никакие runtime-поведение, CLI-флаги, конфигурация или API не меняются; это
  документационный change, описывающий уже существующее поведение точнее и
  полнее.
