## ADDED Requirements

### Requirement: Флаг --retry-from
CLI команда `run` ДОЛЖНА поддерживать флаг `--retry-from <agent-name>`.

#### Scenario: Перезапуск с указанного агента
- **КОГДА** пользователь запускает `ai-team run --retry-from coder`
- **ТОГДА** пайплайн ДОЛЖЕН пропустить всех агентов до `coder` (включительно: analyst, architect — пропущены)
- **И** начать выполнение с агента `coder`
- **И** артефакты пропущенных этапов НЕ ДОЛЖНЫ удаляться

### Requirement: Проверка входных артефактов при retry
Система ДОЛЖНА проверять наличие всех необходимых входных артефактов перед запуском с указанного этапа.

#### Scenario: Ошибка при отсутствии артефактов
- **КОГДА** пользователь запускает `--retry-from deployer`, но артефакты tester отсутствуют
- **ТОГДА** система ДОЛЖНА вывести ошибку: `missing artifacts from previous stage: tester`
- **И** НЕ ДОЛЖНА запускать агента

### Requirement: Повторный запуск завершённого этапа
`--retry-from` ДОЛЖЕН позволять перезапускать уже завершённый агент.

#### Scenario: Retry на последнем агенте
- **КОГДА** deployer завершился с ошибкой
- **И** пользователь запускает `ai-team run --retry-from deployer`
- **ТОГДА** пайплайн ДОЛЖЕН запустить только deployer заново
- **И** старые артефакты deployer ДОЛЖНЫ быть перезаписаны
