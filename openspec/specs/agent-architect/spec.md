## ДОБАВЛЕННЫЕ Требования

### Требование: Architect создаёт технический дизайн
Агент Architect ДОЛЖЕН прочитать продуктовую спецификацию и создать технический дизайн.

#### Сценарий: Architect создаёт артефакты
- **КОГДА** Architect запускается
- **ТОГДА** он ДОЛЖЕН прочитать `.ai-team/artifacts/{feature}/specs/`
- **И** создать `.ai-team/artifacts/{feature}/design.md`
- **И** создать `.ai-team/artifacts/{feature}/tasks.md`

### Требование: Содержимое технического дизайна
Технический дизайн ДОЛЖЕН содержать архитектуру, модели данных, API-контракты и план реализации.

#### Сценарий: Секции дизайна
- **КОГДА** Architect создаёт design.md
- **ТОГДА** он ДОЛЖЕН включать:
  - Обзор архитектуры
  - Модели данных / схемы
  - API-контракты
  - Разбивку на компоненты
  - Выбор технологий
  - Задачи по реализации в tasks.md

### Требование: Список задач
tasks.md ДОЛЖЕН содержать чеклист задач по реализации.

#### Сценарий: Формат задач
- **КОГДА** Architect создаёт tasks.md
- **ТОГДА** каждая задача ДОЛЖНА быть чекбоксом markdown: `- [ ] Описание задачи`
- **И** задачи ДОЛЖНЫ быть упорядочены по зависимостям

### Требование: Промпт architect
Промпт architect ДОЛЖЕН содержать требования к структуре design.md и tasks.md.

#### Сценарий: Структура design.md
- **КОГДА** architect создаёт design.md
- **ТОГДА** design.md ДОЛЖЕН содержать: затронутые репозитории и компоненты, выбранное техническое решение, изменения по файлам или модулям, изменения контрактов и данных, риски и способы их снижения, порядок реализации

#### Сценарий: Структура tasks.md
- **КОГДА** architect создаёт tasks.md
- **ТОГДА** tasks.md ДОЛЖЕН содержать: зависимый чеклист задач, каждая задача — чекбокс markdown

#### Сценарий: Обнаружение ошибок
- **КОГДА** architect обнаруживает ошибку в requirements
- **ТОГДА** architect ДОЛЖЕН вернуть BLOCKED
