## Purpose

Спецификация определяет нормативное поведение capability `agent-coder`.

## Requirements
### Requirement: Coder реализует код по техническому дизайну
Агент Coder MUST прочитать технический дизайн и реализовать код в целевом проекте.

#### Scenario: Coder запускается
- **КОГДА** Coder запускается
- **ТОГДА** он MUST прочитать `.ai-team/artifacts/{feature}/design.md`
- **И** прочитать `.ai-team/artifacts/{feature}/tasks.md`
- **И** реализовать код в дереве исходников целевого проекта
- **И** написать unit-тесты для нового кода

#### Scenario: Без отклонений от спецификации
- **КОГДА** Coder реализует
- **ТОГДА** он MUST NOT добавлять фичи или поведение, не описанные в дизайне и specs

### Requirement: Coder использует AgentCLI runtime
Coder MUST делегировать фактическую запись кода OpenCode CLI.

#### Scenario: Реализация через OpenCode
- **КОГДА** Coder запускается
- **ТОГДА** система MUST подготовить промпт с контекстом из design.md и tasks.md
- **И** запустить новую сессию `opencode run`, прикрепив промпт временным файлом mode 0600 через `--file`
- **И** MUST NOT использовать `--resume` или `--continue`
- **И** OpenCode MUST записать фактические файлы исходников

### Requirement: Контроль мутаций Coder
Контроллер MUST независимо проверить, что Coder создал атрибутированные текущей попытке изменения только в разрешённых путях.

#### Scenario: Coder не изменил проект
- **КОГДА** runtime завершился с кодом 0, но snapshot проекта не изменился
- **ТОГДА** попытка MUST завершиться ошибкой
- **И** последующие этапы MUST NOT запускаться
