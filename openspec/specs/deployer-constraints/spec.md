## Purpose

Ограничения схемы и исполнения controller-owned delivery.

## Requirements

### Requirement: Exact-file staging
Delivery executor MUST использовать массив нормализованных workspace-relative файлов без glob и MUST отказать при заранее staged index.

#### Scenario: План содержит каталог или control path
- **КОГДА** plan содержит каталог, `.git/`, `.ai-team/`, traversal или дублирующийся путь
- **ТОГДА** plan validation MUST завершиться ошибкой до staging

#### Scenario: Staged set отличается от plan
- **КОГДА** результат `git add -- <exact files>` не совпадает с утверждённым набором
- **ТОГДА** executor MUST отменить собственный staging и MUST NOT создавать commit

### Requirement: Protected branch
Executor MUST запрещать push в `main`, `master` и объявленную base branch.

#### Scenario: Запуск с default branch
- **КОГДА** текущая ветка является base branch
- **ТОГДА** executor MUST создать или выбрать утверждённую feature branch до commit

### Requirement: Метаданные commit и PR
Commit message MUST быть одной строкой не длиннее 120 символов и MUST NOT содержать AI-атрибуцию; PR body MUST быть не длиннее 700 символов.

#### Scenario: Запрещённая атрибуция
- **КОГДА** commit message содержит `Generated by`, `AI-authored` или `Co-authored-by`
- **ТОГДА** plan MUST быть отклонён
