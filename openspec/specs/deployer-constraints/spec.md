## Purpose

Ограничения deployer — строгие правила форматирования коммитов и PR.

## Requirements

### Requirement: Формат коммита
Deployer ДОЛЖЕН соблюдать строгий формат коммит-сообщения.

#### Scenario: Формат commit message
- **КОГДА** deployer создаёт коммит
- **ТОГДА** commit message ДОЛЖЕН начинаться с номера задачи (если есть)
- **И** содержать краткое описание изменения
- **И** быть не длиннее 10 слов после номера задачи
- **И** НЕ содержать упоминаний или атрибуции агента

#### Scenario: Без атрибуции
- **КОГДА** deployer формирует commit message
- **ТОГДА** сообщение НЕ ДОЛЖНО содержать "Generated by", "AI-authored", "Co-authored-by" или подобные футеры

### Requirement: Формат PR
Deployer ДОЛЖЕН соблюдать ограничения на описание PR.

#### Scenario: Длина PR
- **КОГДА** deployer создаёт PR
- **ТОГДА** описание PR ДОЛЖНО быть не длиннее 700 символов
- **И** ДОЛЖНО кратко объяснять: что изменено, зачем, как проверялось

### Requirement: Проверка условий перед деплоем
Deployer ДОЛЖЕН проверять review.md и test-report.md перед выполнением.

#### Scenario: Условия не выполнены
- **КОГДА** review.md НЕ содержит APPROVED или test-report.md НЕ содержит PASS
- **ТОГДА** deployer ДОЛЖЕН вернуть BLOCKED статус
- **И** описать: "Деплой заблокирован: {причина}"

### Requirement: Все артефакты на русском языке
Deployer ДОЛЖЕН создавать PR description на русском языке.

#### Scenario: Язык PR
- **КОГДА** deployer создаёт PR
- **ТОГДА** описание PR ДОЛЖНО быть на русском языке
