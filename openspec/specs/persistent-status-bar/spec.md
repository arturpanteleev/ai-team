## Purpose

Спецификация определяет нормативное поведение capability `persistent-status-bar`.

## Requirements
### Requirement: Status bar внизу терминала
Система MUST показывать персистентную строку состояния внизу терминала.

#### Scenario: Status bar содержит проект, фичу, агента и прогресс
- **КОГДА** пайплайн запущен
- **ТОГДА** внизу терминала MUST отображаться строка: `[ai-team] <project> | <feature> | <agent> (N/M)`
- **И** строка MUST оставаться видимой после вывода логов
- **И** строка MUST обновляться после каждого завершённого этапа

### Requirement: Status bar не затирается логами
Система MUST использовать ANSI escape для сохранения/восстановления позиции курсора.

#### Scenario: Логи выше status bar
- **КОГДА** агент выводит логи (через `fmt.Print`, `fmt.Println`)
- **ТОГДА** статус-бар MUST оставаться внизу терминала
- **И** логи MUST отображаться выше статус-бара

### Requirement: Отключение status bar
Status bar MUST отключаться, если stdout не является терминалом.

#### Scenario: Нет status bar при перенаправлении
- **КОГДА** stdout перенаправлен в файл или pipe
- **ТОГДА** status bar MUST NOT отображаться
